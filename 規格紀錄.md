### UI 規格更新

- `TestCase 管理` 頁面移除標題下方說明文字。
- TestCase 列表改為表格呈現，欄位為 `Product`、`Module`、`Case`、`Priority`、`update`，其中 `update` 以 `yyyy-mm-dd` 格式顯示。
- 列表操作統一提供 `編輯`、`修改`、`刪除` 按鈕。
- 頁尾移除 `簡約風格 TestCase 管理系統 / SQLite + Python` 文案。
- `新增 TestCase` 表單中的 `Case 標題` 改為 `Case`。
- `新增 TestCase` 與 `編輯 TestCase` 表單新增 `Remark` 欄位。
- `Remark` 欄位排序調整為表單與預覽側邊面板的最後一列。
- `Priority` 預設值為 `Medium`。
- 新增、修改、刪除提示改為懸浮通知，顯示 3 秒後自動消失。
- TestCase 資料新增 `updated_at` 欄位，用於列表 `update` 顯示與更新時間追蹤。
- `TestCase 管理` 頁面 Header 中央新增搜尋列，可對 `Case` 進行模糊查詢。
- `TestCase 管理` 預覽側邊面板新增 `Remark` 顯示，並可點擊 `修改` 後直接在側邊面板編輯所有欄位；此時才會顯示 `保存` 按鈕，保存成功後會回到唯讀模式並隱藏 `保存` 按鈕。
- `TestCase 管理` 列表與預覽側邊面板唯讀模式中，欄位值若為空會顯示 `N/A`。
- `新增 TestCase` 與預覽側邊面板編輯模式的必填欄位為 `Product`、`Module`、`Case`、`Step`、`Expected Result`，欄位標籤會顯示紅色 `*`，且失焦時若為空會在欄位下方顯示紅字 `請輸入欄位`。
- `新增 TestCase` 頁面點擊 `保存` 時，會一次檢核所有必填欄位；若有空值，所有對應欄位都會同時顯示紅字 `請輸入欄位`，並於頁面中間上方顯示 `請先完成必填欄位` 提示。
- `TestCase 管理` 主內容區寬度已加大，並針對 `Product`、`Module`、`Case` 欄位調整為不自動換行，降低新建資料在列表中折行的情況。
- `TestCase 管理` 預覽側邊面板寬度由原本約三分之一調整為五分之二，以提供更充足的預覽與編輯空間。
